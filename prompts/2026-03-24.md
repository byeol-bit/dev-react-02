## 2026-03-24

### Q1
질문: 라이브러리 설치과정에서 나타난 에러가 어떤것들이 있는지 말해줘.

답변:
- 핵심 에러는 `ERESOLVE unable to resolve dependency tree` 입니다.
- 현재 프로젝트는 `react@19.2.4`를 사용 중인데, 설치하려던 `react-beautiful-dnd@13.1.1`의 peer dependency가 `react ^16.8.5 || ^17.0.0 || ^18.0.0`까지만 지원해서 충돌이 발생했습니다.
- 따라서 npm이 의존성 트리를 확정하지 못해 설치를 중단했습니다.
- 로그에는 우회 옵션으로 `--force` 또는 `--legacy-peer-deps`가 제시되어 있지만, 이는 잠재적으로 깨진 조합을 허용할 수 있다는 경고가 함께 표시됩니다.

### Q2
질문: react를 다운그레이드 하려면 어떻게 하면 될까?

답변:
- 현재 충돌 원인(`react-beautiful-dnd`가 React 18까지만 지원)을 기준으로, `react`와 `react-dom`을 함께 `18.x`로 맞추는 방식이 가장 안전합니다.
- 권장 순서:
  1) 기존 설치물 정리: `node_modules`, `package-lock.json` 삭제
  2) 버전 지정 설치: `npm install react@18.3.1 react-dom@18.3.1`
  3) 나머지 패키지 설치 재시도
- 타입스크립트 사용 시 `@types/react`, `@types/react-dom`도 18 버전대로 맞추는 것이 좋습니다.

### Q3
질문: `'ILogItem'이(가) 선언은 되었지만 해당 값이 읽히지는 않았습니다.` 에러를 고쳐줘.

답변:
- `ILogItem`를 값처럼 잘못 초기화하던 코드(`logArray: ILogItem[]`)를 제거하고, 상태 타입(`TLoggerState`)의 타입 annotation으로 사용하도록 수정했습니다.
- `initialState`를 `const initialState: TLoggerState = { logArray: [] }` 형태로 바꿔서 `ILogItem` import가 실제 타입 위치에서 사용되게 했습니다.
- 검증: `npm run lint -- "src/store/slices/loggerSlice.ts"` 실행 결과 exit code 0으로 통과했습니다.
